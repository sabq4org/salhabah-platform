# منصة صلهبه التعليمية

## الاسم الرسمي
ابتدائية صلهبه الأولى وروضة صبيا الثالثة — صبيا

## الاسم التقني
`salhabah-platform`

## وصف مختصر
منصة داخلية متكاملة لإدارة العمليات المدرسية في ابتدائية صلهبه الأولى وروضة صبيا الثالثة، تشمل الإدارة، الطالبات، المعلمات، الإداريات، الفصول، الجداول، الدرجات، الشهادات، الإعلانات، التواصل الداخلي، الإشراف التربوي، والتقارير.

## الهدف
توحيد بيانات المدرسة وعملياتها اليومية في نظام واضح وآمن، مع صلاحيات دقيقة لكل فئة مستخدمة، وسجل متابعة للعمليات الحساسة.

## المرحلة الدراسية
- **الروضة:** KG1 (مستوى أول) / KG2 (مستوى ثاني) / KG3 (تمهيدي)
- **الابتدائي:** الصف الأول → الصف السادس

## أقسام المشروع
- `01-docs`: وثائق الفكرة والمتطلبات والصلاحيات وخارطة الطريق.
- `02-design`: الهوية البصرية، الألوان، الأفكار، والتصاميم الأولية.
- `03-data-model`: تصور البيانات الأساسية للمنصة.
- `04-modules`: تقسيم الوظائف حسب الوحدات الرئيسية.
- `05-ui-pages`: وصف الصفحات والشاشات.
- `06-security`: الخصوصية، الصلاحيات، النسخ الاحتياطي، وسجل التدقيق.
- `07-reports`: تقارير الطالبات، المعلمات، الفصول، الدرجات، والحضور.
- `08-future-features`: أفكار مستقبلية مثل بوابة ولي الأمر وتطبيق الجوال.
- `AGENTS.md`: دليل تشغيلي للوكيل الذكي عند فتح المشروع في جلسة AI جديدة.

## التشغيل المحلي

```bash
# 1. ضعي قاعدة البيانات + سرّ التوقيع في .env.local (انسخي من .env.example)
#    DATABASE_URL=postgresql://...neon...?sslmode=require
#    AUTH_SECRET=<48 byte hex — openssl rand -hex 48>

# 2. ثبّتي الحزم
npm install

# 3. طبّقي السكيما على القاعدة (مرة واحدة)
npx drizzle-kit push --force

# 4. ابذري بيانات تجريبية (اختياري)
npm run db:seed

# 5. خادم تطوير على بورت 3001 (لتفادي تعارض المنصات الأخرى)
npm run dev -- -p 3001
```

افتحي [http://localhost:3001](http://localhost:3001).

## حسابات الدخول التجريبية (بعد `db:seed`)

| اسم المستخدم | كلمة المرور | الدور |
|---|---|---|
| `admin` | `admin1234` | مديرة المدرسة (مها عبدالفتاح) |
| `staff` | `staff1234` | وكيلة المدرسة |
| `teacher` | `teacher1234` | معلمة (نورة السبيعي) |
| `guardian` | `guardian1234` | ولي أمر |

> **مهم:** هذه حسابات تجريبية لبيئة التطوير فقط. غيّري كل كلمات المرور
> في الإنتاج، وفعّلي إجبار تغيير كلمة المرور عند أول دخول.

## أوامر مفيدة

```bash
npm run dev                         # تطوير
npm run build && npm start          # بناء وتشغيل production
npm run db:push                     # تطبيق السكيما
npm run db:studio                   # واجهة استكشاف drizzle
npm run db:seed                     # بذر بيانات تجريبية (idempotent)
npm run db:export                   # تصدير شامل لكل الجداول
npm run db:import-students -- file  # استيراد طالبات من JSON
npm run db:import-teachers          # استيراد معلمات من JSON
npm run db:purge-fake               # حذف بيانات seed التجريبية فقط
```

## النسخ الاحتياطي والاستعادة

من الإعدادات → **النسخ الاحتياطي** (المديرة فقط):
- **تصدير:** ينزّل ملف JSON واحد يحوي كامل بيانات المنصة.
- **استعادة:** يستبدل كل البيانات بمحتوى ملف نسخة سابقة (بعد كتابة جملة تأكيدية).
  حساب المديرة الحالي يُحفظ تلقائياً لتفادي القفل خارج النظام.

## الأمان وسياسات الحساب

- سياسة كلمة المرور: ١٠ أحرف فأكثر، حروف كبيرة وصغيرة ورقم ورمز خاص.
- يُجبَر المستخدم على تغيير كلمة المرور عند أول دخول وعند إعادة تعيينها (`/change-password`).
- قفل تلقائي للحساب بعد ٥ محاولات فاشلة لمدة ١٥ دقيقة.
- تحديد معدّل لمحاولات الدخول حسب IP + اسم المستخدم خلال نافذة ١٥ دقيقة.
- حماية CSRF لنموذج الدخول عبر رمز مزدوج (Cookie + حقل مخفي) + التحقق من Origin.
- جميع نماذج Server Actions تتحقق من الصلاحية عبر `assertPermission`.
- الكوكيز: `sal_session` (HttpOnly) + `sal_csrf` (لتأكيد double-submit).

### متغيرات البيئة الأمنية
- `AUTH_SECRET` (إلزامي للإنتاج).
- `AUTH_ALLOWED_ORIGINS` (اختياري): قائمة Origins إضافية مسموح بها مفصولة بفواصل.
- `AUTH_ALLOW_MISSING_ORIGIN=1` (اختياري، غير موصى به).
